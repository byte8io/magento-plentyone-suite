# Mage2Plenty v3.5 - Placeholder SKU Quarantine & Self-Healing Stock Sync

## Highlights

- **Placeholder SKU quarantine** — regex skips `NEW-XXXX` rows until renamed.
- **Self-healing stock pre-processors** — mapping re-resolved every run.
- **Variation rename cascade** — re-pends stock on upstream SKU change.
- **`plenty:stock:reconcile` command** — diagnoses and repairs drift.

## What's New

- **Placeholder SKU Pattern** profile field — recommended `^NEW-\d+$`.
- **`SourceAssignmentCheck`** post-processor — NOTICE on missing source row.
- **Reservation orphan join** — anchored on `plenty_variation_id`, not SKU.
- **`Inventory::invalidateMappingByVariationIds()`** — bulk re-pend helper.
- **`variation.number` is authoritative** — `variation.sku` deprecated.

## Bug Fixes

- Fixed placeholder SKUs polluting `catalog_product_entity` mapping.
- Fixed stale `plenty_stock_entity` mapping surviving variation renames.
- Fixed orphan-reservation false positives during reconciliation.
- Fixed `SKIPPED` rows incorrectly stamped `COMPLETE` after unresolved mapping.
- Fixed price attribute "None" option not persisting due to null value.

## Post-Upgrade Notes

Set **Placeholder SKU Pattern** to `^NEW-\d+$` on each item-import profile.

Run `bin/magento plenty:stock:reconcile` once to surface pre-existing drift; review before re-running with `--fix`.

Migrate any third-party reads of `VariationInterface::SKU` / `getSku()` to `getNumber()` — wrappers stay for now, removed in next major.
