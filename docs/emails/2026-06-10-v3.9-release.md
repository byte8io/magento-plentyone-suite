# Mage2Plenty v3.9 - Payment-Safe Order Export & Storage-Location Stock Cleanup

## What's New

- **Payment-safe order export** - Export now waits a configurable settling period (default 60s) before syncing an order to PlentyONE, giving async gateways (Mollie/Klarna) time to finalise payment so you don't export unpaid orders. A wiring fix ensures the new guard actually engages.
- **New-order email gate** - Order import can now suppress Magento's native new-order and invoice confirmation emails for imported PlentyONE orders, via a "Send New Order Email" toggle (default off).
- **Storage-location stock cleanup** - New `plenty:stock:location:correct` CLI zeroes stock at a specific PlentyONE storage location, fixing quantities double-counted across storage locations within a warehouse. Includes `--dry-run`, a test-slice filter, batching and a live progress bar.

## Post-Upgrade Notes

No schema changes, but two defaults change behaviour on upgrade:

- **Order export waits 60 seconds** after order creation before exporting. Set **Export Settling Period** to `0` on the export profile schedule to restore immediate export.
- **The new-order email gate defaults to off**, so imported orders no longer trigger Magento's native new-order email unless you enable **Send New Order Email** on the import action config.

Run `setup:di:compile` and `cache:flush` after upgrading.
