# Mage2Plenty v3.7 - Order Import Hardening & Stock Delta-Mode Fix

## What's New

- **Shipping profile fallback** - Order import now falls back to the line items when the order-level shipping profile is missing, so the shipping method and tracking carrier still resolve.
- **Case-insensitive payment fallback** - The import payment method (including the configured fallback) is now normalized, so a casing typo like `PlentyONE` no longer triggers "The requested Payment Method is not available" and blocks the order.
- **Order tag constants** - Added `TAGS` / `TAG_ID` to the order REST API interface for order-tag handling.
- **Stock delta-mode fix** - Corrected the `bookIncomingItems` URL specifier (`%S` → `%s`) that broke incoming bookings in v3.6 delta-mode stock export.

## Post-Upgrade Notes

No configuration changes required. If you run **delta-mode stock export**, restart the consumer after upgrade: `bin/magento queue:consumers:start plenty.stock.export.processor`.
