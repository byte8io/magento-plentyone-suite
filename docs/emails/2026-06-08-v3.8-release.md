# Mage2Plenty v3.8 - Offline-Capture Safety & Stock Import Efficiency

## What's New

- **Offline-capture safety** - Imported PlentyONE orders are flagged for offline capture, so invoicing one in Magento never re-charges the customer through a live payment gateway.
- **Shipping profile fallback** - Order import now falls back to the order-item properties for the shipping profile ID when the order-level property has none.
- **Skip unmanaged products** - Stock import now skips products configured with "Manage Stock = No", suppressing both physical-stock and reservation sync for them.
- **Resilient source assignment** - Unsupported stock types (e.g. `preorder` reservations) are logged and skipped instead of erroring the import row.
- **Reservation table index** - A new `sku` index on `inventory_reservation` removes a full table scan that ran on every stock import / order export lookup.

## Post-Upgrade Notes

No configuration changes required. `setup:upgrade` adds the `inventory_reservation` SKU index via declarative schema (a single `ALTER TABLE`), so run the upgrade during a low-traffic window.
