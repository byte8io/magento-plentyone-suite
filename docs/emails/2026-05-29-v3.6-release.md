# Mage2Plenty v3.6 - Stock Export Profile with Channel-Safe Delta Corrections

## Highlights

- **Stock Export Profile** — Magento → PlentyONE physical-stock sync, event-driven + cron + MQ.
- **Delta correction mode** — `bookIncomingItems` / `bookOutgoingItems` preserves Plenty-side Amazon, eBay, channel deductions.
- **Bootstrap policies** — `absolute_then_delta` default aligns day-zero, then preserves channels forever.
- **Shipment-decrement producer** — closes the hour-long gap where shipped stock wasn't reaching Plenty.
- **Magento 2.4.8 compatibility fix** — disabled `gc_collect_cycles()` in batch loops across stock / customer / item services.

## What's New

- **`plenty_stock_export_queue`** table with five event producers + hourly drift reconciler safety net.
- **`Stock Correction Mode`** profile field: `absolute` (default) or `delta` (channel-safe via reasonId 100 / 200).
- **`Delta Bootstrap Policy`** profile field — `absolute_then_delta` / `skip_and_seed` / `reconcile_to_cache`.
- **`metadata.plenty_bookings`** audit trail — captures Plenty booking `id` and `orderNo` per push.
- **MQ topic `plenty.stock.export.processor`** + cron job `plenty_stock_export` (group `plenty_stock_profile`).
- **CLI commands**: `plenty:stock:export[:add|:purge|:drift]` with `--status`, `--all`, `--older-than`, `--force` flags.
- **`bookOutgoingItems`** REST API method + W3C `deliveredAt` + `supplierId` payload constants.

## Bug Fixes

- Fixed `processed_at` never being set on successful exports (overall-status was always `complete`, not `success`).
- Fixed per-SKU status corruption — sibling SKU failures in a batch no longer flip successful rows to ERROR / WARNING.
- Fixed shipment-driven physical decrements bypassing the export producers; Plenty stayed stale until the next drift sweep.
- Fixed `gc_collect_cycles()` in batch cleanup causing infinite re-fetch loops on Magento 2.4.8 resource collections.
- Fixed stock export admin grid showing empty rows (stale `QueueListingDataProvider` virtualType replaced with concrete data provider against the new table).
- Fixed `Cannot instantiate interface` error on "Queue for Export" mass-action (missing `ExportStockProcessor\PublisherInterface` DI preference).

## Post-Upgrade Notes

Set **Stock Correction Mode** on each export profile. Leave `absolute` (default) for stores with no Plenty-side channels. Flip to `delta` for any store where Plenty sells the same SKUs on Amazon, eBay, or similar — and leave bootstrap policy on the new `absolute_then_delta` default.

Seed the queue once at onboarding: `bin/magento plenty:stock:export:add --all` (drift reconciler needs `plenty_stock_entity` cache populated, which is typically empty on first install — `--all` is the right tool for the very first push). Use `plenty:stock:export:drift` for ongoing maintenance.

Start the MQ consumer for async high-priority pushes: `bin/magento queue:consumers:start plenty.stock.export.processor`. The cron job `plenty_stock_export` runs the periodic drain (admin-configurable schedule via the `plenty_stock_profile` group). Drift reconciler runs hourly at `15 * * * *` as the safety net.
